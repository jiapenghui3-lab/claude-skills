# `/tutor` Design Notes

`/tutor` 的设计目标是把 Claude Code 从一次性问答工具扩展成长期学习伙伴。它的关键不是“讲得更长”，而是让学习过程具备结构、反馈、记忆和调度。

## 核心设计

### 1. 文件系统记忆

学习不是只存在于聊天记录里，而是沉淀到当前工作目录的 `learning/` 文件夹中。

这让 `/tutor` 可以跨会话恢复进度，也让学习记录可以被搜索、备份、复盘和迁移。

### 2. 学生画像

`_profile.md` 用来记录学习者长期稳定的信息，包括背景、目标、偏好、强弱项和错误模式。

这让系统可以逐步减少无效解释，提高每次教学的命中率。

### 3. 进度图谱

`_progress.md` 不只是课程列表，而是一个轻量知识图谱。它记录概念之间的前置关系，也记录每个节点的掌握状态。

这样系统可以判断什么时候推进，什么时候补依赖，什么时候复习。

### 4. 掌握度评估

`/tutor` 使用三个维度判断学习质量：

- 概念复述准确度
- 类比迁移能力
- 变式应用能力

只会复述不代表学会，能迁移和应用才更接近真正掌握。

### 5. 自适应节奏

系统会根据学生反馈和答题表现调整节奏：

- 太简单就加速
- 太难就降级
- 卡住就换角度
- 状态差就减负
- 学太久就建议休息

## 和普通提示词的区别

普通提示词通常解决“这次怎么回答”。`/tutor` 解决的是“一个主题如何长期学下去”。

它的价值在于把教学拆成可重复执行的闭环：

```text
评估 -> 教学 -> 作答 -> 评分 -> 更新画像 -> 更新进度 -> 复习/推进
```

这也是它最像系统，而不是最像提示词的地方。
