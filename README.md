# Claude Skills

个人 Claude Code slash command 集合。

这是一个面向 Claude Code 的自定义 skill / slash command 仓库，目标不是简单堆几条提示词，而是把可复用的工作流、角色设定、文件系统记忆、学习进度管理和任务执行规范沉淀成一套可以随时调用的个人能力库。

目前仓库的主力 command 是 `/tutor`：一个基于 Bloom 2-Sigma 思想设计的自适应一对一学习系统。它会把 Claude Code 从“回答问题的 AI”升级成“会建档、会追踪、会复盘、会调难度、会记住你怎么学”的私人导师。你可以拿它学编程、数学、英语、产品、投资、写作、AI 工具、框架源码，甚至任何你想拆开的复杂主题。

如果普通 AI 教学像是在临时讲一节课，`/tutor` 更像是给你开了一个长期学习工作台：它知道你现在学到哪里、哪里薄弱、哪些概念已经掌握、哪些知识点该复习、下一步应该推进还是回头补依赖。它不只负责“讲明白”，还负责“确认你真的学会”。

## 为什么它看起来有点猛

`/tutor` 的核心不是一段花哨人设，而是一整套学习闭环：

- 它会为每个主题自动创建独立学习目录，避免不同主题混在一起。
- 它会维护学生画像，记录你的背景、目标、学习风格、认知强项、常见错误模式。
- 它会维护学习进度，记录当前状态、完成模块、薄弱知识点和后续路线。
- 它会生成知识图谱，用 DAG 追踪概念之间的前置依赖和掌握状态。
- 它会根据答题表现动态调节难度，而不是机械地一节一节往下讲。
- 它会把复习机制内置进学习流程，每个新文件都包含上一节的回顾题。
- 它会逐题评分，解释扣分原因，并给出综合掌握度。
- 它会在你状态不好、卡太久、学太久的时候主动降速或建议休息。
- 它支持新对话恢复学习，读取历史记录后从上次断点继续。

简单说，它不是“AI 老师提示词”，而是一个小型学习操作系统。

## `/tutor` 能做什么

### 1. 任意主题启动

你可以直接输入：

```text
/tutor React
/tutor Python
/tutor 线性代数
/tutor 英语语法
/tutor 产品经理方法论
/tutor 大模型应用开发
```

如果你只输入 `/tutor`，它会先问你要学什么。

### 2. 自动建立学习档案

每个主题都会拥有自己的学习空间：

```text
learning/
└── <主题>/
    ├── _profile.md
    ├── _progress.md
    ├── 01_入门评估.md
    ├── 02_xxx.md
    └── ...
```

这意味着你学 React 的进度不会污染 Python，你学数学的薄弱点也不会和英语混在一起。每个主题都是独立项目，每次都能恢复现场。

### 3. 学生画像系统

`_profile.md` 会记录：

- 行业背景
- 编程基础
- 学习目标
- 学习风格
- 认知特征
- 强项与弱项
- 有效学习策略
- 无效学习策略
- 错误模式追踪
- 老师人设偏好

这让 `/tutor` 不再用一套固定话术教所有人，而是逐渐贴合你的理解方式。

### 4. 进度与知识图谱系统

`_progress.md` 会记录：

- 当前正在学什么
- 当前掌握度
- 下一步学习动作
- 已完成模块表格
- 待强化知识点
- 间隔重复队列
- Mermaid 知识图谱
- 节点状态表

它不只是“保存历史”，而是在维护一个可继续推进的学习地图。

### 5. 掌握度评分

系统会从三个维度评估你是不是真的懂了：

| 维度 | 含义 |
|------|------|
| A | 概念复述准确度 |
| B | 类比迁移能力 |
| C | 变式应用能力 |

综合掌握度计算方式：

```text
综合掌握度 = A * 0.3 + B * 0.3 + C * 0.4
```

然后根据结果决定下一步：

| 掌握度 | 动作 |
|------|------|
| ≥ 80% | 进入新概念 |
| 60% - 80% | 同概念换角度继续巩固 |
| < 60% | 回溯前置依赖 |

这套机制让它像一个会判断学习质量的教练，而不是一个只会继续输出下一章内容的文档生成器。

### 6. 自适应学习节奏

你可以随时输入特殊指令：

| 指令 | 效果 |
|------|------|
| `太简单了` | 下一个文件跳升 2 个难度等级 |
| `太难了` | 回到最近答对的层级，换一种方式讲 |
| `给我看全景` | 从 `_progress.md` 生成知识图谱可视化 |
| `换个方式讲` | 同概念换一个完全不同角度解释 |
| `我想跳到xxx` | 检查前置依赖后决定是否跳转 |
| `复习` | 从间隔重复队列提取薄弱点生成复习文件 |
| `测验我` | 从已完成模块随机抽题 |
| `总结我学了什么` | 生成一页纸学习总结 |
| `我今天状态不好` | 降难度、减内容、减少题目 |

它不是死板课程表，而是能跟着你状态变化的学习系统。

### 7. 严格但温柔的人设

`/tutor` 不是无限夸夸型助手。它的老师人设是“略严格但温柔”：答得好会开心，答偏了会指出问题，卡住了会耐心引导，但不会用空洞鼓励糊弄你。

这种设定的意义是：学习过程里既要有情绪反馈，也要有真实纠偏。你会感觉自己不是在读教程，而是在跟一个认真负责、记得你学习轨迹的人一起推进。

### 8. 面向长期学习

很多 AI 学习对话的问题是：这次讲完，下次就断了。`/tutor` 的设计重点正好相反：它默认学习是一个长期过程。

新对话恢复时，它会读取：

- `_profile.md`
- `_progress.md`
- 最近 2 个学习文件
- 学生反馈和答题记录

然后告诉你：

```text
我已读取你的记录。上次学到了 xxx，掌握度 xx%，薄弱点是 xxx。我们从这里继续。
```

这就是它最值钱的地方：不是临场聪明，而是能连续工作。

## 适合用来学什么

只要是可以拆成概念、练习、反馈和复习的主题，都适合交给 `/tutor`。

| 类型 | 示例 |
|------|------|
| 编程语言 | Python、JavaScript、TypeScript、Go、Rust |
| 前端开发 | React、Vue、Next.js、CSS、浏览器原理 |
| 后端开发 | Node.js、数据库、API 设计、缓存、消息队列 |
| AI 应用 | Prompt Engineering、RAG、Agent、OpenAI API |
| 计算机基础 | 数据结构、算法、操作系统、网络、编译原理 |
| 数学 | 线性代数、概率论、微积分、离散数学 |
| 产品与商业 | 产品经理、增长、商业模式、用户研究 |
| 写作表达 | 结构化写作、演讲、文案、英文写作 |
| 考试复习 | 章节拆解、错题复盘、薄弱点强化 |

## 设计原则

这个仓库里的 command 会尽量遵循以下原则：

- 能沉淀文件，就不要只停留在对话里。
- 能追踪进度，就不要每次从零开始。
- 能评分反馈，就不要只给泛泛建议。
- 能拆概念，就不要一次塞太多内容。
- 能复习薄弱点，就不要盲目追求新知识。
- 能适应用户状态，就不要机械推进。
- 能查官方来源，就不要编造技术定义。
- 能长期复用，就不要写成一次性提示词。

## 使用方式

把 `commands/` 目录下的 `.md` 文件复制到本机 Claude Code 命令目录：

```bash
cp commands/*.md ~/.claude/commands/
```

然后重启 Claude Code，即可使用对应的 slash command。

例如：

```text
/tutor TypeScript
```

## 推荐使用姿势

建议为每个学习项目单独开一个工作目录，让 `/tutor` 在当前目录下创建 `learning/` 文件夹。这样学习记录会天然跟项目绑定，也方便之后回看、迁移和备份。

如果你想长期学习一个主题，不要只让它讲知识点。更好的方式是：

1. 先让 `/tutor <主题>` 创建学习档案。
2. 回答 `01_入门评估.md`。
3. 把答案发回给 Claude Code。
4. 让它逐题评分并生成下一份学习文件。
5. 学一段时间后输入 `给我看全景` 或 `复习`。

这样它会越来越了解你，学习路径也会越来越贴身。

## 一句话总结

`/tutor` 把 Claude Code 变成了一个带长期记忆、知识图谱、掌握度评估、间隔复习、自适应难度和人格反馈的私人学习系统。它不是让你“听懂一段解释”，而是逼近一个更硬核的目标：让你真的掌握一个主题。
