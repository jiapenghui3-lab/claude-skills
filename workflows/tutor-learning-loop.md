# Tutor Learning Loop

`/tutor` 的核心工作流是一条长期学习闭环。

```text
启动主题
  -> 读取或创建学习档案
  -> 生成学习文件
  -> 学生作答
  -> 逐题评分
  -> 更新学生画像
  -> 更新学习进度
  -> 判断推进、巩固或回溯
  -> 生成下一份学习文件
```

## 1. 启动主题

输入 `/tutor <主题>` 后，系统先检查当前目录是否已有对应主题记录。

## 2. 读取或创建档案

已有主题：读取 `_profile.md` 和 `_progress.md`。

新主题：创建学习目录并生成入门评估。

## 3. 生成学习文件

每份学习文件最多 2 个核心概念，内容控制在可消化范围内。

每道题后面紧跟回答框，避免学生读完一大段后不知道该怎么反馈。

## 4. 评分与反馈

系统从概念复述、迁移能力、应用能力三个维度判断掌握度，并逐题解释扣分原因。

## 5. 更新记录

系统更新：

- 学生画像
- 错误模式
- 当前模块
- 掌握度
- 薄弱点
- 知识图谱
- 复习队列

## 6. 决策下一步

| 情况 | 下一步 |
|------|--------|
| 掌握度高 | 进入新概念 |
| 掌握度中等 | 同概念换角度 |
| 掌握度低 | 回溯前置依赖 |
| 状态不好 | 降难度、减题量 |
