# `/tutor` Session Example

这个示例展示 `/tutor` 的理想使用方式：先建档，再评估，再反馈，再推进。

## 1. 启动

```text
/tutor TypeScript
```

系统会检查当前目录是否存在：

```text
learning/TypeScript/
```

如果不存在，会创建：

```text
learning/TypeScript/
├── _profile.md
├── _progress.md
└── 01_入门评估.md
```

## 2. 学生作答

学生打开 `01_入门评估.md`，按每道题下面的回答框作答，然后把答案发回给 Claude Code。

## 3. 导师评分

`/tutor` 会逐题分析：

- 概念复述是否准确
- 是否能迁移到相似场景
- 是否能处理变式问题
- 哪些地方只是背下来了但没理解
- 哪些错误会影响后续学习

然后给出综合掌握度。

## 4. 生成下一份学习文件

如果掌握度高，进入新概念。

如果掌握度中等，换角度继续巩固。

如果掌握度低，回到前置依赖重新讲。

## 5. 长期循环

每次学习都会更新：

- `_profile.md`
- `_progress.md`
- 最近学习文件
- 待强化知识点
- 知识图谱

最终形成一个越学越懂你的私人学习系统。
