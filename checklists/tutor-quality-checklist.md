# Tutor Quality Checklist

用于检查 `/tutor` 生成的学习文件和反馈是否符合预期。

## 学习文件

- [ ] 是否最多只包含 2 个核心概念
- [ ] 是否控制在学生能消化的长度
- [ ] 是否每道题后都有回答框
- [ ] 是否包含上一份学习文件的复习题
- [ ] 是否没有把内部规则暴露给学生
- [ ] 是否避免了牵强类比
- [ ] 是否没有编造技术定义

## 评分反馈

- [ ] 是否逐题评分
- [ ] 是否解释扣分原因
- [ ] 是否给出综合掌握度
- [ ] 是否指出具体薄弱点
- [ ] 是否决定下一步是推进、巩固还是回溯

## 进度维护

- [ ] 是否更新 `_profile.md`
- [ ] 是否更新 `_progress.md`
- [ ] 是否更新待强化知识点
- [ ] 是否更新知识图谱
- [ ] 是否记录当前卡点

## 教学体验

- [ ] 是否语气自然
- [ ] 是否严格但不打击人
- [ ] 是否没有过度夸奖
- [ ] 是否在学生状态不好时主动降速
- [ ] 是否在连续卡住时建议休息
